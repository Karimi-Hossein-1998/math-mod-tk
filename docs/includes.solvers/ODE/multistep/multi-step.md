# Multi-Step Methods

In this document we will discuss the algorithm and implementation of the multi-step methods for solving ordinary differential equations (ODEs). Multi-step methods are a class of numerical methods that use multiple previous points to calculate the next point in the solution. This can lead to higher accuracy and efficiency compared to single-step methods, especially for stiff problems.

The class of multi-step methods includes both explicit and implicit methods. Explicit multi-step methods, such as Adams-Bashforth methods, use only previously computed values to calculate the next value. Implicit multi-step methods, such as Adams-Moulton methods, involve solving an equation that includes the next value, which can provide better stability properties.

There are different kinds of multi-step methods, including:

1. Adams-Bashforth methods (explicit)
2. Adams-Moulton methods (implicit)
3. Backward differentiation formulas (BDF) (implicit)
4. Nystrom methods (explicit)
5. Predictor-corrector methods (combination of explicit and implicit/Adams-Bashforth-Moulton)
6. etc.

In the following sections, we will go through the algorithmic details of these methods, their implementation in code, and examples of their usage. We will also discuss the stability and convergence properties of multi-step methods, as well as their advantages and disadvantages compared to single-step methods.

**Notes:**

- Fixed-step assumption: derivations assume constant step size $h = t_{n+1}-t_n$ unless stated otherwise.
- Coefficient ordering: coefficient arrays are ordered earliest→latest so $\beta_0$ corresponds to $f(t_{n-s+1},y_{n-s+1})$.
- Implementation: coefficients are generated by `abm.py` and stored in [ABM-Coefs](../../../../includes/miscellany/ABM-Coefs/); a C++ namespace is provided in [abm-coefs.hpp](../../../../includes/namespaces/abm-coefs.hpp). See the miscellany and namespace docs for details.
- Vector-valued ODEs: all interpolations and integrations apply component-wise for vector `y`; see the [Lagrange interpolator documentation](../../../includes.interpolators/lagrange-interpolator.md) for vector-valued function interpolation.

## Adams-Bashforth method

As mentioned previously, the Adams-Bashforth method is an explicit multi-step method that uses the previous $s$ values of the function to compute the next value. In next section we will derive the coefficients for the Adams-Bashforth method and provide the formulas for the first few orders.

### AB derivation

Imagine the following equation:

$$ \displaystyle y' = f(t,y), $$

$$ \displaystyle y(t_0) = y_0. $$

We know that given some data pairs of $(\text{input},\text{output})$ for an unknown function we can re-create an estimator function.

We formulate the ODE as

$$\displaystyle y' = f(t,y),\qquad y(t_0)=y_0.$$

To derive Adams–Bashforth coefficients we interpolate the map

$$\displaystyle \varphi(t)=f\big(t,y(t)\big)$$

using the last $s$ known values of $\varphi$ at the nodes

$$\displaystyle t_{n-s+1},\; t_{n-s+2},\; \dots,\; t_n.$$

An $s$-step Adams–Bashforth method (explicit) then has the standard form

$$\displaystyle y_{n+1} = y_n + h\sum_{j=0}^{s-1}\beta_j\,\varphi_{n-s+1+j},$$

where we use the short-hand $\varphi_m=\varphi(t_m)=f(t_m,y_m)$ and $h=t_{n+1}-t_n$. Here $\beta_0$ multiplies $\varphi_{n-s+1}$, $\beta_1$ multiplies $\varphi_{n-s+2}$, and so on.

The coefficients $\beta_j$ are obtained by integrating the Lagrange interpolant of $\varphi$ over $[t_n,t_{n+1}]$. Let $L_j(t)$ denote the Lagrange basis polynomial associated with node $t_{n-s+1+j}$ (so $j=0,\dots,s-1$). Then

$$\displaystyle \beta_j=\frac{1}{h}\int_{t_n}^{t_{n+1}}L_j(t)\,\mathrm{d}t.$$

Perform the affine change of variable

$$\displaystyle t=t_n+h\tau,\qquad \tau\in[0,1],\qquad \mathrm{d}t=h\,\mathrm{d}\tau.$$

Introduce the scaled node locations (so that $\beta_0$ corresponds to the stored node $t_{n-s+1}$):

$$\displaystyle \tau_j=\frac{t_{n-s+1+j}-t_n}{h}=j-(s-1),\qquad j=0,\dots,s-1,$$

and define the Lagrange basis on the $\tau$-variable as

$$\displaystyle L_j(\tau)=\prod_{m\neq j}\frac{\tau-\tau_m}{\tau_j-\tau_m}=\prod_{m\ne j}\frac{\tau-(m-(s-1))}{j-m}.$$

For numerical stability when evaluating interpolants in code, prefer the barycentric form of the Lagrange interpolant (see the Lagrange interpolator documentation).

With this notation

$$\displaystyle \beta_j=\int_0^1 L_j(\tau)\,\mathrm{d}\tau.$$

Below are the canonical Adams–Bashforth coefficients written so that $\beta_0$ corresponds to the earliest stored node $t_{n-s+1}$:

- s=1 (AB1): $\;\beta_0=1$.
- s=2 (AB2): $\;\beta_0=-\tfrac12,\;\beta_1=\tfrac32$.
- s=3 (AB3): $\;\beta_0=\tfrac{5}{12},\;\beta_1=-\tfrac{16}{12},\;\beta_2=\tfrac{23}{12}$.
- s=4 (AB4): $\;\beta_0=-\tfrac{9}{24},\;\beta_1=\tfrac{37}{24},\;\beta_2=-\tfrac{59}{24},\;\beta_3=\tfrac{55}{24}$.

Each s-step Adams–Bashforth method has order $s$ (i.e., local truncation error ~ $O(h^{s+1})$), determined by the degree of the interpolating polynomial used for $\varphi$.

For higher orders (s > 4) and additional coefficient tables including Adams–Moulton methods, see the [ABM-Coefs](../../../includes/miscellany/ABM-Coefs/) directory.

Practical remarks:

- Starting values: an s-step multistep method requires the previous $s-1$ solution values; obtain them with a one-step method (e.g. a Runge–Kutta integrator) or lower-order multistep rules.
- Stability: Adams–Bashforth methods are explicit and have restricted stability regions; Adams–Moulton formulas are implicit and often used as correctors in predictor–corrector schemes.
- Consistency and order: the order equals the degree of the interpolating polynomial used for $\varphi$, and the coefficients above follow from integrating that polynomial over $[0,1]$ under the affine map.

**Note:** This method has been implemented in [ab-solver.hpp](../../../../includes/solvers/ODE/multistep/ab-solver.hpp).

## Adams-Moulton method

As mentioned earlier the Adams-Moulton method is an implicit multi-step method that uses the previous $s$ values of the function and the next value to compute the next value. In next section we will derive the coefficients for the Adams-Moulton method and provide the formulas for the first few orders.

### AM derivation

The Adams-Moulton method is closely related to Adams–Bashforth but is **implicit** it uses the unknown $y_{n+1}$ in the step formula. An $s$-step Adams–Moulton method has the form

$$\displaystyle y_{n+1} = y_n + h\sum_{j=0}^{s}\beta_j\,\varphi_{n-s+1+j},$$

where $\varphi_m = f(t_m, y_m)$ and $h=t_{n+1}-t_n$. Compared to Adams–Bashforth (which uses $s$ nodes), the Adams–Moulton method includes $s+1$ nodes: $t_{n-s+1}, \ldots, t_n, t_{n+1}$. The term $\beta_s \varphi_{n+1}$ depends on the unknown $y_{n+1}$, requiring implicit solution.

The coefficients $\beta_j$ follow the same Lagrange-interpolant derivation as Adams–Bashforth, but now integrate the interpolant over $[t_n, t_{n+1}]$ using all $s+1$ nodes. Using the affine substitution $t = t_n + h\tau$ ($\tau \in [0,1]$) with scaled nodes

$$\displaystyle \tau_j=\frac{t_{n-s+1+j}-t_n}{h}=j-(s-1),\qquad j=0,\ldots,s,$$

the Lagrange basis polynomials are defined as

$$\displaystyle L_j(\tau)=\prod_{m\neq j}\frac{\tau-\tau_m}{\tau_j-\tau_m}=\prod_{m\ne j}\frac{\tau-(m-(s-1))}{j-m},$$

the coefficients are

$$\displaystyle \beta_j=\int_0^1 L_j(\tau)\,\mathrm{d}\tau,$$

where $L_j(\tau)$ are the Lagrange basis polynomials on the scaled nodes. Each $s$-step Adams–Moulton method has order $s+1$ (one order higher than the corresponding Adams–Bashforth).

Coefficients for $s=1,...,10$ are tabulated in the [ABM-Coefs](../../../includes/miscellany/ABM-Coefs/) directory.

**Solution of the implicit equation:** To compute $y_{n+1}$ from the implicit formula, use a nonlinear solver (e.g., Newton–Raphson) applied to the residual

$$\displaystyle r(y)=y - y_n - h\Big(\sum_{j=0}^{s-1}\beta_j f(t_{n-s+1+j}, y_{n-s+1+j}) + \beta_s f(t_{n+1}, y)\Big),$$

so the unknown appears only in the final term $\beta_s f(t_{n+1},y)$. Alternatively use a predictor–corrector approach: first predict $y_{n+1}^{(0)}$ with Adams–Bashforth, then iteratively correct using the Adams–Moulton formula until convergence.

## Predictor–Corrector methods

By combining an explicit method (predictor) with an implicit method (corrector), we obtain a **predictor–corrector scheme** that avoids the cost of solving a nonlinear equation iteratively (e.g., via Newton–Raphson), while maintaining the stability and accuracy benefits of implicit methods.

### Adams-Bashforth-Moulton predictor-corrector scheme

The Adams–Bashforth–Moulton predictor–corrector uses AB to predict and AM to correct. The formulation is:

$$ \displaystyle \hat{y}_{n+1} = y_n + h\sum_{j=0}^{s-1}\beta_j^{AB}\,\varphi_{n-s+1+j} \quad \text{(predictor)}, $$

And then we use this predicted value $\hat{y}_{n+1}$ in the Adams-Moulton formula to get $\hat{\varphi}_{n+1}=f(t_{n+1};\hat{y}_{n+1})\approx\varphi_{n+1}$ and then get the corrected value as:

$$ \displaystyle y_{n+1} = y_n + h\sum_{j=0}^{s}\beta_j^{AM}\,\varphi_{n-s+1+j} \quad \text{(corrector)}, $$

where $\beta_j^{AB}$ and $\beta_j^{AM}$ are the Adams–Bashforth and Adams–Moulton coefficients (see [ABM-Coefs](../../../includes/miscellany/ABM-Coefs/) directory for tabulated values). In the corrector step, $\varphi_{n-s+1}, \ldots, \varphi_n$ are known values from previous steps, and $\varphi_{n+1} = f(t_{n+1}, \hat{y}_{n+1})$ is newly evaluated from the predicted value $\hat{y}_{n+1}$. This avoids solving a nonlinear system directly while retaining the accuracy and stability of implicit methods.

**PECE:** What described above is also called a *PECE* scheme (Predict->Evaluate->Correct->Evaluate) since we first predict, then evaluate the function at the predicted value, then correct using the implicit formula, and finally evaluate again at the corrected value if needed.

**Order of the scheme:** The order of the predictor-corrector scheme is typically determined by the order of the corrector (Adams-Moulton) since it provides the final value. For example, if we use AB2 as the predictor and AM2 as the corrector, the overall scheme will have third-order accuracy. ($s$-step AM corrector is of order $s+1$ while $s$-step AB predictor is of order $s$, so the corrector's order dominates).

**PECECE...:** Corrector can be applied multiple times (iteratively) to further refine the solution, using the latest corrected value to update $\varphi_{n+1}$ and recompute $y_{n+1}$ until convergence. This is called a *PE$\text{(CE)}^m$* scheme, where $m$ is the number of corrector iterations. This is implemented in [abm-solver.hpp](../../../../includes/solvers/ODE/multistep/abm-solver.hpp)

**Convergence:** If we have a *PE$\text{(CE)}^m$* scheme, the convergence criterion is generally considered to be:

    1. $\|\hat{y}_{n+1}^{(k+1)}-\hat{y}_{n+1}^{(k)}\| < \epsilon \qquad k\in\{1,2,\ldots,m-1\}.$
    2. If $ k=m $ then $y_{n+1}=\hat{y}_{n+1}^{(m)}$ regardless of the difference between the last two iterations.

Recommended practical parameters: choose $\epsilon$ in the range $[10^{-12},10^{-6}]$ depending on problem scale, and cap the number of corrector iterations to 3--5 to avoid costly inner loops.

**Stability:** The stability region of a *PE$\text{(CE)}^m$* scheme is dominated by the corrector (Adams–Moulton). For best results, use the same s-step for both predictor and corrector to balance stability and efficiency; in practice, one or two correction iterations (PECE or PE(CE)²) suffice.

### Extensions and variants

Other predictor–corrector pairs (e.g., Adams–Bashforth–Adams–Bashforth, or Runge–Kutta–Adams–Moulton) and additional refinements (e.g., step-size control, local error estimation) are common in adaptive multistep solvers. There is a catch with adaptive methods and that is if you don't use evenly spaced time points you need to **re-calculate** the coefficients ($\beta_{j}^{AB}$, $\beta_{j}^{AM}$) at each step, which can be computationally expensive. In practice, many adaptive multistep solvers use a fixed set of coefficients corresponding to a reference step size and adjust the step size while keeping the same coefficients, which can lead to reduced accuracy if the step size changes significantly.

## Examples and Usage

Here we will demonstrate the usage of the implemented solvers within [ab-solver.hpp](../../../../includes/solvers/ODE/multistep/ab-solver.hpp) and [abm-solver.hpp](../../../../includes/solvers/ODE/multistep/abm-solver.hpp) with some example ODEs. We will compare the results to known analytical solutions or reference numerical solutions to verify correctness and illustrate the behavior of the methods in proper *Examples* which will be available within [Examples](../../../../Examples/) directory.

1. **Function Name**: `adams_bashforth_solver`
    - **Description**: Solves an ODE using the explicit Adams–Bashforth method (order 1–10).
    - **Signatures**:
          - `SolverResults adams_bashforth_solver(const SolverParameters& params)` — full `SolverResults` including time points.
          - `dMatrix adams_bashforth_solver(MyFunc deriv, const dVec& y0, double t0, double t1, double dt, int order)` — convenience wrapper returning the solution matrix.
    - **Notes**: The solver bootstraps the first `order-1` values with `rk4_solver` internally.

Example — using `SolverParameters` (recommended):

    ```C++
    // Define ODE: y' = -y (exponential decay)
    MyFunc f = [](double t, const dVec& y) -> dVec {
        dVec r(y.size());
        for (size_t i = 0; i < y.size(); ++i) r[i] = -y[i];
        return r;
    };

    SolverParameters params;
    params.derivative = f;
    params.initial_conditions = {1.0};
    params.t0 = 0.0;
    params.t1 = 5.0;
    params.dt = 0.1;
    params.order = 4; // AB4

    SolverResults res = adams_bashforth_solver(params);
    // Access results: res.solution, res.time_points
    ```

Example — using convenience wrapper:

    ```C++
    dMatrix sol = adams_bashforth_solver(f, {1.0}, 0.0, 5.0, 0.1, 4);
    // sol[row] is y at time t0 + row*dt
    ```

2. **Function Name**: `adams_bashforth_moulton_solver`
    - **Description**: Predictor–corrector ABM solver (order 1–10) combining an explicit Adams–Bashforth predictor with an Adams–Moulton corrector.
    - **Signatures**:
          - `SolverResults adams_bashforth_moulton_solver(const SolverParameters& params)` — full `SolverResults`.
          - `dMatrix adams_bashforth_moulton_solver(MyFunc deriv, const dVec& y0, double t0, double t1, double dt, int order, int corrector_iters)` — convenience wrapper.
    - **Notes**: The `params.iterations` field controls the number of corrector iterations (PE(CE)^m). The solver bootstraps with `rk4_solver` for the first `order-1` steps.

Example — ABM with `SolverParameters`:

    ```C++
    SolverParameters p_abm;
    p_abm.derivative = f;
    p_abm.initial_conditions = {1.0};
    p_abm.t0 = 0.0;
    p_abm.t1 = 5.0;
    p_abm.dt = 0.1;
    p_abm.order = 3;        // use 3-step AB predictor / 3-step AM corrector (AM order = 4)
    p_abm.iterations = 2;   // PE(CE)^2

    SolverResults r_abm = adams_bashforth_moulton_solver(p_abm);
    // r_abm.solution, r_abm.time_points
    ```

Example — ABM convenience wrapper:

    ```C++
    dMatrix sol_abm = adams_bashforth_moulton_solver(f, {1.0}, 0.0, 5.0, 0.1, 3, 2);
    ```

Remarks:

- Both solvers return solutions in row-major time order: the first row is the initial condition at `t0`.
- For vector-valued systems the solvers operate component-wise; the `dMatrix` has `dVec` rows for each time point.
- If you require error estimates or adaptive step control, consider single-step adaptive solvers (see adaptive solvers documentation). The multistep adaptive solvers/error estimates are yet to be implemented.
- The AB and ABM implementations rely on coefficient tables in the `abm_coefs` namespace; to add new orders regenerate coefficients with `abm.py` and update `abm-coefs.hpp`.
